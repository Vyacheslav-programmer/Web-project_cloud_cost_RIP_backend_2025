#### Цель работы
Выбор варианта-темы на весь курс, знакомство с разработкой бэкенда и разработка дизайна для 3 страниц системы оценки стоимости облачного хостинга.

#### Задание
Создание дизайна приложения в `figma`, базовая шаблонизация в Django для `тарифов облачного хостинга`, развертывание `Minio`.

Требуется разработать дизайн трех страниц приложения в `figma`, повторив при этом основную стилистику уже существующего популярного ресурса (3 цвета с точными кодами, hoover, форму карточек), связанного с облачным хостингом. Данные по теме также нужно искать реальные.

Создание базового интерфейса, состоящего из трех страниц. Первая для просмотра списка `тарифов облачного хостинга` в виде карточек (плиткой в 2-4 столбца) с наименованием, ценой, характеристиками (CPU, RAM, SSD) и картинкой, а также отдельным элементом-карточкой текущего `расчёта стоимости` (корзина) с указанием количества `тарифов` в ней. Это количество вычисляется в контроллере-обработчике. При клике по карточке происходит переход на вторую страницу с подробной информацией о `тарифе` (описание, характеристики, цена). А при нажатии на текущий `расчёт` по его id открывается страница просмотра состава `расчёта`, в которой есть поля самого `расчёта`, каждый его `тариф` отображается ниже отдельной карточкой друг под другом, поля `тарифа` вывести слева направо в 1 строку и справа для каждого из них поле `м-м` (количество месяцев/основной тариф/комментарий), которое в последующих лабораторных сможет менять `клиент`. Также в `расчёте` или в еще одном поле `м-м` должно быть поле результата вычислений (пока указано готовое значение).

В `header` приложения добавить только кнопку/иконку `Домой`, она для навигации на страницу `тарифов`. Добавить поле input для фильтрации на сервере списка `тарифов` по одному из полей (наименование, цена, количество ядер), отображаемых на странице (по умолчанию отображать все). Поле поиска должно сохраняться после запроса. Всего в приложении должно быть 3 GET запроса и две модели-коллекции. Все данные для обеих страниц нужно брать прямо из двух коллекций (массива тарифов и словаря расчёта), без использования БД. Никаких массивов внутри отдельного `тарифа` (соблюдать атомарность и 1НФ), но можно и нужно писать большие текстовые описания. Никакого редактирования, только поиск и просмотр. Без `JavaScript`.

В приложении должны быть использованы стили, для каждого элемента списка подгружается свое изображение. Изображения `тарифов` отображаются на всех трех страницах. Изображения хранятся в `Minio`, наименование изображения указано ключом на латинице, хранится в модели отдельным полем. Разработать стиль приложения, который будет применяться далее в последующих лабораторных по фронтенду. `CSS` вынести в отдельный файл.
